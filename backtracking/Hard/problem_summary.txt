1. SUDOKU SOLVER
   Link: https://leetcode.com/problems/sudoku-solver/
   
   Problem Explanation:
   - Given a 9×9 Sudoku board with some cells filled and others empty (0)
   - Fill empty cells such that:
     * Each row contains digits 1-9 exactly once
     * Each column contains digits 1-9 exactly once
     * Each 3×3 subgrid contains digits 1-9 exactly once
   - Find ANY valid solution
   
   Key Idea:
   - Find empty cell, try digits 1-9
   - For each digit, validate 3 constraints (row, col, 3×3 box)
   - If valid, place digit and recurse
   - If recursion succeeds, return true (found solution)
   - If recursion fails, backtrack (try next digit)
   - Base case: no empty cells remain (solved)
   
   Constraint Validation:
   - Vertical: scan entire column for digit
   - Horizontal: scan entire row for digit
   - 3×3 Box: scan the 3×3 subgrid containing the cell
     * Start row = (row/3)*3, Start col = (col/3)*3
     * Check 9 cells in that subgrid
   
   Simple: Fill empty cells with valid digits, validate all 3 Sudoku rules
   
   Time Complexity: O(9^(N)) where N = number of empty cells, typically 9^40+
   Space Complexity: O(1) if we ignore recursion stack, O(N) for stack depth
   
   Key Challenges:
   - Multiple simultaneous constraints (hardest aspect)
   - Proper early pruning is critical for performance
   - One mistake in any constraint breaks solution

2. KNIGHT'S TOUR (Validation)
   Link: https://leetcode.com/problems/valid-knight-tour/
   File: nightsTour.cpp
   
   Problem Explanation:
   - Given an NxN grid with values 0 to N²-1
   - Validate if these values represent a VALID KNIGHT'S TOUR
   - Knight's tour: knight moves through all squares exactly once
   - Knight moves in L-shape: 2 squares in one direction, 1 perpendicular
   
   Key Idea:
   - Start at grid[0][0], which should have value 0
   - Recursively check if all 8 possible knight moves lead to next expected value
   - 8 knight moves: 
     * (-2, +1), (-1, +2), (+1, +2), (+2, +1)
     * (+2, -1), (+1, -2), (-1, -2), (-2, -1)
   - Check boundary and correct next value
   - Base case: when expected value == N²-1 (all squares visited)
   - Any path works, just need ONE valid sequence
   
   Simple: From current position, verify one of 8 knight moves has next sequence value
   
   Time Complexity: O(8^(N²)) worst case, but with proper structure much better
   Space Complexity: O(N²) for recursion depth
   
   Key Challenge:
   - Must validate specific tour sequence in grid
   - Knights move is unusual (2+1 L-shape)

3. N KNIGHTS PROBLEM
   Link: Not on LeetCode (custom problem)
   File: nNights.cpp
   
   Problem Explanation:
   - Place N knights on an N×N chessboard
   - No two knights can attack each other
   - Knights attack cells that are a knight's move away (L-shape)
   - Find ALL valid placements
   
   Key Idea:
   - Similar to N-Queens but with knight attack pattern instead of queen
   - For each cell, check if placing knight here is safe
   - isSafe() checks all 8 possible knight move positions
   - If they all have no other knights, position is safe
   - Place knight, recurse to next row/cell
   - Backtrack and try next position
   
   Knight Attack Validation:
   - Check 8 L-shaped positions around current cell
   - Positions: (r±2, c±1) and (r±1, c±2)
   - Verify boundary and no knight at any attacking position
   
   Simple: Place knights row by row, verify no other knight in L-shaped attack pattern
   
   Time Complexity: O(N!) - similar to N-Queens but checks different constraint
   Space Complexity: O(N) for recursion depth
   
   Key Differences from N-Queens:
   - Knight moves in L-shape (not rows/cols/diagonals)
   - 8 possible attack positions (not entire row/col/diagonals)
   - Can place knights in any empty square (not just one per row)

HARD PROBLEMS: KEY PATTERNS & TECHNIQUES

┌─────────────────────────────────────────────────────────────────────────────┐
│ MULTI-CONSTRAINT SATISFACTION                                               │
│ ─────────────────────────────────────────────────────────────────────────  │
│ Sudoku: 3 simultaneous constraints (row, col, 3×3)                         │
│ N-Queens: 3 constraints (row, col, diagonals)                              │
│ N-Knights: 1 constraint (L-shape attacks)                                  │
│                                                                             │
│ → CRITICAL: Check ALL constraints before placing                           │
│ → Use early pruning to eliminate impossible states                         │
│ → Order constraints by most selective first                                │
└─────────────────────────────────────────────────────────────────────────────┘

EFFICIENCY STRATEGIES FOR HARD PROBLEMS:

1. EARLY PRUNING
   - Check constraints BEFORE recursing, not after
   - Eliminate impossible branches immediately
   - Reduces search space exponentially

2. CONSTRAINT ORDERING
   - Apply most restrictive constraints first
   - Example: In Sudoku, a cell with few valid options first

3. MEMOIZATION (when applicable)
   - Cache results of expensive constraint checks
   - Avoid rechecking same board state

4. BIT MANIPULATION (optional)
   - Represent board state efficiently
   - Faster constraint checking with bitwise operations

5. SYMMETRY BREAKING
   - Skip symmetric placements
   - Reduces redundant computation

COMPARISON: MEDIUM VS HARD

MEDIUM PROBLEMS:
- Single type of constraint (parentheses, subsets, etc.)
- ~10-100 million comparisons for typical input
- Can be solved with straightforward backtracking
- Time limit easily met with basic implementation

HARD PROBLEMS:
- Multiple simultaneous constraints
- ~10^15+ comparisons possible for worst case
- REQUIRE efficient constraint checking and early pruning
- Naive implementation will timeout
- Need optimization beyond basic backtracking

SOLVING HARD PROBLEMS: STEP-BY-STEP APPROACH

1. UNDERSTAND THE STATE SPACE
   - What are we trying to place/fill?
   - What's the maximum size? (9×9 Sudoku, N×N board)

2. IDENTIFY ALL CONSTRAINTS
   - List every rule that must be satisfied
   - Determine which are most restrictive

3. DESIGN VALIDATION APPROACH
   - How to efficiently check each constraint?
   - Can constraints be checked incrementally?

4. IMPLEMENT BACKTRACKING
   - Choose an ordering (row-by-row, cell-by-cell, etc.)
   - Apply constraints in order of restrictiveness
   - Return early on failure, explore on success

5. OPTIMIZE
   - Profile which constraints take longest
   - Cache results where possible
   - Consider alternative orderings

6. TEST
   - Start with smallest inputs
   - Verify solution correctness
   - Check edge cases

PREREQUISITES FOR HARD PROBLEMS:
✓ Deep understanding of backtracking pattern
✓ Comfort with recursive thinking
✓ Ability to design efficient constraint checks
✓ Experience with at least 5-10 medium problems
✓ Understanding of time/space complexity analysis

WHEN TO USE BACKTRACKING vs ALTERNATIVES:
✓ Use Backtracking when:
  - Need to FIND solution (any or all)
  - Problem has constraints to satisfy
  - State space is discrete and manageable
  - Other approaches (BFS, DP) are too slow

✗ Don't use for:
  - Simple enumeration (use loops)
  - Problems solvable by greedy algorithms
  - Problems with exponential state space requiring pruning
    (consider approximation or other approaches)
