BITWISE OPERATORS - MEDIUM LEVEL PROBLEMS
==========================================

1. IS POWER OF TWO
   Link: https://leetcode.com/problems/power-of-two/
   
   Problem Explanation:
   - Determine if a given number is a power of 2
   - Examples: 8 (2^3), 16 (2^4) are powers of 2; 6, 10 are not
   
   Key Idea:
   - Powers of 2 have exactly one bit set in binary (1000, 10000, etc)
   - Use trick: n & (n-1) == 0 → if true, n is power of 2
   - Why: n-1 flips all bits after the single set bit, so AND becomes 0
   - Example: 8 = 1000, 7 = 0111 → 1000 & 0111 = 0000
   
   Simple: Check if n & (n-1) equals 0. If yes, it's a power of 2.

2. REVERSE INTEGER
   Link: https://leetcode.com/problems/reverse-integer/
   
   Problem Explanation:
   - Reverse the digits of a signed 32-bit integer
   - If reversed integer overflows 32-bit range, return 0
   - Examples: 123 → 321, -123 → -321, 120 → 21
   
   Key Idea:
   - Extract each digit using modulo (x % 10)
   - Build reversed number by: res = res * 10 + digit
   - Check overflow before adding each digit to prevent wraparound
   - Check if res > MAX/10 or (res == MAX/10 && digit > MAX%10) before positive overflow
   - Check if res < MIN/10 or (res == MIN/10 && digit < MIN%10) before negative overflow
   
   Simple: Extract digits one by one, check overflow conditions, and rebuild the number.

3. SUM OF TWO INTEGERS
   Link: https://leetcode.com/problems/sum-of-two-integers/
   
   Problem Explanation:
   - Add two integers without using the + operator
   - Must use bitwise operations to implement addition
   - Examples: 5 + 3 = 8, -2 + 5 = 3
   
   Key Idea:
   - XOR (^) gives sum without carry: a ^ b
   - AND (&) gives carry: (a & b) << 1
   - Repeat: a = a ^ b (sum), b = (a & b) << 1 (carry)
   - Continue until carry (b) becomes 0
   - Use temp variable to store carry before updating a
   
   Simple: Use XOR for sum and AND<<1 for carry. Loop until no carry left.

4. POW(X, N)
   Link: https://leetcode.com/problems/powx-n/
   
   Problem Explanation:
   - Compute x raised to the power n (x^n) efficiently
   - Handle edge cases: negative exponents, zero base, integer overflow
   - Examples: 2^10 = 1024, 2.0^(-2) = 0.25
   
   Key Idea:
   - Use fast exponentiation with bit manipulation approach
   - Instead of multiplying x by itself n times, use binary representation of n
   - For each bit in n: if bit is 1, multiply result by current power of x
   - Square x for each bit position (x → x^2 → x^4 → x^8, etc)
   - Time complexity: O(log n) instead of O(n)
   - Example: 2^13 = 2^(1101) = 2^8 * 2^4 * 2^1
   
   Simple: Use bit manipulation of exponent. For each bit, multiply if bit=1, then square base.

5. XOR SUM
   
   Problem Explanation:
   - Calculate XOR of all elements in an array
   - XOR of all elements where duplicates exist will cancel them out
   
   Key Idea:
   - Initialize result = 0
   - XOR each element: result ^= arr[i]
   - XOR is commutative and associative: order doesn't matter
   - Useful for finding missing/extra numbers in pairs
   
   Simple: Start with 0 and XOR every element. Pairs cancel, leaving unique elements.
