BITWISE OPERATORS - EASY LEVEL PROBLEMS
========================================

1. BITWISE OPERATORS INTRODUCTION
   
   Problem Explanation:
   - Learn fundamental bitwise operations: AND, OR, XOR, NOT, left shift, right shift
   - Understand how these operations work on binary representations
   
   Key Idea:
   - AND (&): Both bits must be 1 → Result is 1
   - OR (|): At least one bit is 1 → Result is 1
   - XOR (^): Bits are different → Result is 1
   - NOT (~): Flip all bits (two's complement for negative)
   - Left shift (<<): Multiply by 2^n
   - Right shift (>>): Divide by 2^n
   
   Simple: Understand how bits interact. AND for common bits, OR for combined bits, XOR for differences.

---

2. GET ITH BIT
   
   Problem Explanation:
   - For a given number and position i, retrieve/extract the bit at that position
   - Position 0 is the rightmost (least significant) bit
   
   Key Idea:
   - Use right shift to move the desired bit to position 0: num >> i
   - Check the last bit using AND with 1: (num >> i) & 1
   - If result is 1, bit is set; if 0, bit is unset
   
   Simple: Shift the bit to the rightmost position, then check if it's 1 or 0.

---

3. SET BIT
   
   Problem Explanation:
   - Set the bit at position i to 1 in a given number
   - If already 1, no change; if 0, make it 1
   
   Key Idea:
   - Create a mask with 1 at position i: 1 << i
   - Use OR operation to set that bit: num | (1 << i)
   - OR ensures the bit becomes 1 regardless of current value
   
   Simple: Create a mask with 1 at target position, then OR with the number.

---

4. CLEAR ITH BIT
   
   Problem Explanation:
   - Clear/unset the bit at position i to 0 in a given number
   - If already 0, no change; if 1, make it 0
   
   Key Idea:
   - Method 1 (AND): Create mask with 0 at position i, 1s elsewhere: ~(1 << i), then AND
   - Method 2 (XOR): Use XOR to toggle bit if set: (1 << i) to flip at position i
   - Both methods clear the bit effectively
   
   Simple: Create a mask with 0 at target position and XOR or AND with it.

---

5. CLEAR I BITS
   
   Problem Explanation:
   - Clear all bits from position 0 to position i-1 (clear rightmost i bits)
   - Keep bits from position i onwards
   
   Key Idea:
   - Create mask with 0s in rightmost i positions: ~0 << i
   - This creates all 1s shifted left by i positions
   - AND the mask with the number to clear rightmost bits
   
   Simple: Shift all 1s left by i positions to create mask, then AND with number.



6. UPDATE ITH BIT
   
   Problem Explanation:
   - Update/change the bit at position i to a specific value (0 or 1)
   - Works regardless of the bit's current state
   
   Key Idea:
   - Two-step process:
     1. Clear the bit at position i: n & (~(1 << i))
     2. Set it to desired value: cleared_num | (val << i)
   - First clear ensures we're not adding an extra 1
   - Then set based on the value parameter
   
   Simple: First clear the bit, then OR it with the desired value (shifted to position i).

7. COUNT SET BITS
   Link: https://leetcode.com/problems/number-of-1-bits/
   
   Problem Explanation:
   - Count the total number of 1s (set bits) in the binary representation
   - Example: 12 = 1100 → count = 2
   
   Key Idea:
   - Loop through each bit using: n & 1 to check last bit
   - Count it if it's 1
   - Right shift to move to next bit: n >> 1
   - Continue until n becomes 0
   - Time: O(log n) - proportional to number of bits
   
   Simple: Check last bit with & 1, count if set, then right shift to next bit.

8. FIND SINGLE NUMBER
   Link: https://leetcode.com/problems/single-number/
   
   Problem Explanation:
   - In array where every number appears twice except one, find the single number
   - Example: [2, 2, 3, 4, 3, 4, 5] → answer is 5
   
   Key Idea:
   - Use XOR property: a ^ a = 0 and a ^ 0 = a
   - XOR all elements: duplicates cancel each other (2^2^3^4^3^4 = 0), leaving single number
   - Single pass, O(n) time, O(1) space
   - Example: 2^2 = 0, 3^3 = 0, 4^4 = 0, remaining 5
   
   Simple: XOR all elements. Pairs cancel out, leaving the single number.

9. COUNTING BITS
   Link: https://leetcode.com/problems/counting-bits/
   
   Problem Explanation:
   - For a given number n, return array containing count of set bits (1s) in binary representation of each number from 0 to n
   - Example: n = 4 → [0, 1, 1, 2, 1] (0→0bits, 1→1bit, 2→1bit, 3→2bits, 4→1bit)
   
   Key Idea:
   - For each number i from 0 to n, count set bits individually
   - Use (i & 1) to check if last bit is 1
   - Right shift i by 1: (i >> 1) to check next bit
   - Repeat until i becomes 0
   - Time: O(n * log n) - for each number, O(log n) to count bits
   - Space: O(1) excluding output array
   
   Simple: For each number, count 1s by checking last bit and right shifting until zero.

10. REVERSE BITS
   Link: https://leetcode.com/problems/reverse-bits/
   
   Problem Explanation:
   - Reverse the bits of a given 32-bit unsigned integer
   - Example: 00000010100101000001111010011100 → 00111001011110000010100101000000
   
   Key Idea:
   - Iterate through all 32 bits
   - Extract i-th bit from right: (n >> i) & 1
   - Place extracted bit at reversed position (31 - i): bit << (31 - i)
   - OR all bits into result: res | (bit << (31 - i))
   - Continue for all 32 bit positions
   - Time: O(1) - always 32 iterations
   - Space: O(1) - only one integer variable
   
   Simple: Extract bit from position i, place it at position (31-i), repeat for all 32 bits.

11. MISSING NUMBER
   Link: https://leetcode.com/problems/missing-number/
   
   Problem Explanation:
   - Given array containing n distinct numbers from 0 to n (one number is missing), find the missing number
   - Example: [3, 0, 1] → missing number is 2
   
   Key Idea:
   - XOR has useful properties: a ^ a = 0 and a ^ 0 = a
   - XOR all numbers from 0 to n: this gives XOR of all expected numbers
   - XOR all numbers in the array: this gives XOR of all present numbers
   - XOR the two results: duplicate numbers cancel (a ^ a = 0), leaving only the missing number
   - Time: O(n) - two passes through data
   - Space: O(1) - only two variables
   
   Simple: XOR expected numbers (0 to n) with array numbers; missing number survives.
