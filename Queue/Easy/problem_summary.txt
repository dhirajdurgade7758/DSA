1. DEQUE - BASIC OPERATIONS
   Link: https://en.cppreference.com/w/cpp/container/deque
   
   Problem Explanation:
   - Double-ended queue allows insertion/deletion at both front and back
   - Operations: push_back(), push_front(), pop_back(), pop_front()
   - Access: front(), back()
   
   Key Idea:
   - Use STL deque for efficient operations at both ends
   - Time Complexity: O(1) for all operations
   - Space Complexity: O(n) where n is number of elements
   
   Implementation:
   - Create deque, push elements at back and front
   - Remove elements from both ends
   - Access and print front and back elements
   
   Concepts Learned:
   - Double-ended queue basics
   - STL deque container functionality

====================================================================

2. QUEUE - LINKED LIST IMPLEMENTATION
   Link: https://leetcode.com/problems/implement-queue-using-stacks/
   
   Problem Explanation:
   - Implement Queue data structure from scratch using linked list
   - Standard FIFO (First In First Out) operations
   - Operations: push(), pop(), front(), empty()
   
   Key Idea:
   - Use head pointer for front and tail pointer for rear
   - push(): Create new node and add at tail
   - pop(): Remove node from head
   - Handle edge cases: empty queue checks
   
   Implementation:
   - Node class with data and next pointer
   - Queue class with head and tail pointers
   - FIFO insertion at rear, deletion from front
   
   Time Complexity:
   - push: O(1)
   - pop: O(1)
   - front: O(1)
   - empty: O(1)

3. QUEUE USING STL DEQUE
   Link: https://en.cppreference.com/w/cpp/container/queue
   
   Problem Explanation:
   - Implement Queue using STL deque as underlying data structure
   - All standard queue operations: push(), pop(), front(), empty()
   - Simpler than linked list implementation using STL
   
   Key Idea:
   - deque provides efficient O(1) operations at both ends
   - push_back() for enqueue operation
   - pop_front() for dequeue operation
   
   Implementation:
   - Wrapper class around deque
   - push(): calls deque.push_back()
   - pop(): calls deque.pop_front()
   - front(): returns deque.front()
   
   Time Complexity: O(1) for all operations
   Space Complexity: O(n)

====================================================================

4. STACK USING DEQUE
   Link: https://en.cppreference.com/w/cpp/container/stack
   
   Problem Explanation:
   - Implement Stack data structure using deque
   - LIFO (Last In First Out) operations
   - Operations: push(), pop(), top(), empty()
   
   Key Idea:
   - Use deque's push_back() and pop_back() for LIFO behavior
   - Deque is more flexible than vector for stack implementation
   - Can access both ends efficiently
   
   Implementation:
   - Wrapper class around deque
   - push(): deque.push_back()
   - pop(): deque.pop_back()
   - top(): deque.back() returns last element
   
   Time Complexity: O(1) for all operations
   Space Complexity: O(n)

====================================================================

5. STL QUEUE - BASIC USAGE
   Link: https://en.cppreference.com/w/cpp/container/queue
   
   Problem Explanation:
   - Use built-in STL queue container
   - Standard queue interface without custom implementation
   - Quick and efficient queue operations
   
   Key Idea:
   - STL queue is template-based container adapter
   - Works as wrapper over deque by default
   - Perfect for standard FIFO operations
   
   Implementation:
   - Create queue<int> object
   - push(): add element to rear
   - pop(): remove element from front
   - front(): access front element
   - empty(): check if queue is empty
   
   Time Complexity: O(1) for all operations
   Benefits:
   - No need for custom implementation
   - Tested and optimized
   - Easy to use and understand
