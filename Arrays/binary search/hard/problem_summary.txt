1. MEDIAN OF TWO SORTED ARRAYS - LeetCode 4
   Link: https://leetcode.com/problems/median-of-two-sorted-arrays/
   Problem Explanation:
   - Given two sorted arrays of different sizes
   - Find the median of the combined array
   - Median is middle element (if odd length) or average of two middle elements (if even)
   - Example: [1,3] and [2] → median is 2
   - Must have O(log(min(m,n))) time complexity
   
   Key Idea:
   - Always binary search on the smaller array to optimize
   - Partition both arrays such that left half equals right half in size
   - For array A with partition index i, array B has partition index j
   - Partition is valid when: A[i-1] <= B[j] AND B[j-1] <= A[i]
   - Use binary search to find valid partition point
   - If total length is odd, median is max of left partition
   - If total length is even, median is average of max(left) and min(right)
   - Handle edge cases with INT_MIN and INT_MAX for boundary elements
   
   Simple: Binary search on smaller array to partition both arrays. Validate partition with cross-boundary comparisons. Calculate median from partition boundaries.

2. TIME BASED KEY-VALUE STORE - LeetCode 981
   Link: https://leetcode.com/problems/time-based-key-value-store/
   Problem Explanation:
   - Design a data structure that stores key-value pairs with timestamps
   - set(key, value, timestamp): store value for key at given timestamp
   - get(key, timestamp): retrieve value for key with largest timestamp <= given timestamp
   - If no such value exists, return empty string
   - Timestamps are strictly increasing for same key
   
   Key Idea:
   - Use hash map: key → vector of (value, timestamp) pairs
   - Timestamps are naturally sorted due to increasing insertion order
   - For get operation, binary search on the vector of (value, timestamp) pairs
   - Find the largest timestamp <= given timestamp
   - If a valid timestamp found where value_timestamp <= given_timestamp, it's a candidate
   - Continue searching right for even larger valid timestamp
   - Return the value corresponding to largest valid timestamp
   - Time per query: O(log m) where m is number of entries for that key
   
   Simple: Store key → vector of (value, timestamp). For get, binary search to find largest timestamp <= target, return corresponding value.
