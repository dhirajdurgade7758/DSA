1. ROTATE IMAGE - LeetCode 48
Link: https://leetcode.com/problems/rotate-image/

Problem Explanation:

You have an n x n matrix representing an image
Rotate the matrix 90 degrees clockwise
Perform the rotation in-place (do not allocate another 2D matrix)
Return the rotated matrix

Key Idea:

Layer-by-layer rotation approach:
- Process matrix in concentric square layers from outer to inner
- For each layer, rotate a group of 4 elements at a time
- Use 4 positions in the layer: top-left, bottom-left, bottom-right, top-right
- Rotate by moving: bottom-left → top-left → top-right → bottom-right → bottom-left
- Save top-left before rotation, use it at the end to complete the cycle
- Use two pointers (l, r) to track layer boundaries
- For each element at distance i from left: move elements and rotate
- Increment inner pointer, decrement outer pointer to move to next layer

Simple: Process matrix layer by layer, rotate 4 elements at a time by shifting them cyclically around each layer, shrinking boundaries until center is reached.

2. SPIRAL MATRIX - LeetCode 54
Link: https://leetcode.com/problems/spiral-matrix/

Problem Explanation:

You have an m x n matrix
Traverse and return all elements of the matrix in spiral order (clockwise from outside to inside)
Start from top-left, go right → down → left → up, then repeat for inner layers
Return the result as a 1D vector

Key Idea:

Four-boundary pointers approach:
- Use 4 boundaries: srow (start row), erow (end row), scol (start column), ecol (end column)
- Traverse in 4 directions while boundaries are valid:
  * Top: traverse left to right along srow, then increment srow
  * Right: traverse top to bottom along ecol, then decrement ecol
  * Bottom: traverse right to left along erow (skip if srow == erow), then decrement erow
  * Left: traverse bottom to top along scol (skip if scol == ecol), then increment scol
- Continue until all boundaries overlap (srow > erow or scol > ecol)
- Use conditions to avoid revisiting single row/column elements

Simple: Use 4 pointers for boundaries, traverse matrix in spiral pattern (top, right, bottom, left), shrinking boundaries after each direction, skip boundaries when they collapse to prevent duplicates.

3. SET MATRIX ZEROES - LeetCode 73
Link: https://leetcode.com/problems/set-matrix-zeroes/

Problem Explanation:

You have an m x n matrix
If an element is 0, set its entire row and column to 0
Modify the matrix in-place (O(1) space complexity)
Do not use extra space for row/column tracking

Key Idea:

Use first row and column as markers:
- Use matrix[0][c] to mark if column c should be zeroed
- Use matrix[r][0] to mark if row r should be zeroed
- Use a separate rowZero boolean flag to track if first row itself needs zeroing
- Step 1: Scan entire matrix, mark zeros in first row/column and rowZero flag
- Step 2: Update inner matrix (from row 1, col 1) based on first row/column markers
- Step 3: Handle first column separately based on matrix[0][0]
- Step 4: Handle first row separately based on rowZero flag
- This avoids overwriting markers before using them for inner matrix

Simple: Use first row and column as storage for zero markers, track first row separately with a flag, then process inner matrix first, then handle boundaries to avoid destroying marker values.

4. MULTIPLY STRINGS - LeetCode 43
Link: https://leetcode.com/problems/multiply-strings/

Problem Explanation:

You have two non-negative integers represented as strings (num1 and num2)
Multiply the two numbers and return the result as a string
Cannot convert strings to integers directly (too large for standard data types)
Both input strings do not have leading zeros except for "0"

Key Idea:

Grade-school multiplication algorithm using array:
- Create a result array of size n + m (where n, m are lengths of num1, num2)
- Traverse both strings from right to left (backward)
- For each pair of digits, multiply and add to result at correct positions
- Position mapping: product of digits at indices i, j contributes to positions i+j and i+j+1
- pos1 = i + j gets the carry, pos2 = i + j + 1 gets the digit
- Handle carry propagation: sum % 10 is the digit, sum / 10 is the carry
- Convert result array to string, skipping leading zeros
- Time: O(n * m), Space: O(n + m)

Simple: Use array to simulate grade-school multiplication with position-based digit placement, handle carries properly, convert to string while skipping leading zeros.
