1. PERMUTATIONS
   Link: https://leetcode.com/problems/permutations/
   
   Problem Explanation:
   - Given a string or array, generate ALL permutations (all possible arrangements)
   - For "abc": Output is "abc", "acb", "bac", "bca", "cab", "cba"
   - Each element appears exactly once in each permutation
   
   Key Idea:
   - Use choice-explore-unchoose pattern (backtracking)
   - For each position, try each unused element
   - Remove element after recursion, add it back (backtrack)
   - Base case: when all elements are used, print result
   
      "abc"

   Start: ""

   Level 0:
   ""
   ├── a → "a"
   │   ├── b → "ab"
   │   │   └── c → "abc" ✅
   │   └── c → "ac"
   │       └── b → "acb" ✅
   │
   ├── b → "b"
   │   ├── a → "ba"
   │   │   └── c → "bac" ✅
   │   └── c → "bc"
   │       └── a → "bca" ✅
   │
   └── c → "c"
      ├── a → "ca"
      │   └── b → "cab" ✅
      └── b → "cb"
         └── a → "cba" ✅

   Simple: Pick an element → Recurse with remaining → Remove element → Backtrack
   
   Time Complexity: O(n! * n) - n! permutations, each takes O(n) time
   Space Complexity: O(n!) for storing all permutations


2. PERMUTATIONS (VECTOR IMPLEMENTATION)
   Link: https://leetcode.com/problems/permutations/
   File: findPermutationsVector.cpp
   
   Problem Explanation:
   - Same as Permutations but handles numeric arrays with vectors
   - Returns vector<vector<int>> of all permutations
   
   Key Idea:
   - Same choice-explore-unchoose backtracking pattern
   - Use vector.erase() to remove picked element
   - Use vector.insert() to restore during backtracking
   - Collect results in 2D vector
   
      [1,2,3]

   Start: []

   Level 0:
   []
   ├── 1 → [1]
   │   ├── 2 → [1,2]
   │   │   └── 3 → [1,2,3] ✅
   │   └── 3 → [1,3]
   │       └── 2 → [1,3,2] ✅
   │
   ├── 2 → [2]
   │   ├── 1 → [2,1]
   │   │   └── 3 → [2,1,3] ✅
   │   └── 3 → [2,3]
   │       └── 1 → [2,3,1] ✅
   │
   └── 3 → [3]
      ├── 1 → [3,1]
      │   └── 2 → [3,1,2] ✅
      └── 2 → [3,2]
         └── 1 → [3,2,1] ✅

   Simple: Generate all orderings of distinct numbers using backtracking
   
   Time Complexity: O(n! * n)
   Space Complexity: O(n! * n)


3. PERMUTATIONS II (WITH DUPLICATES)
   Link: https://leetcode.com/problems/permutations-ii/
   File: permutationsII.cpp
   
   Problem Explanation:
   - Given an array with duplicates, generate ALL unique permutations
   - For [1,1,2]: Output should have UNIQUE permutations only, not duplicates
   - Key difference: Need to handle duplicate elements smartly
   
   Key Idea:
   - Use unordered_map to track frequency of each unique number
   - For each unique number, if its count > 0, we can use it
   - Decrement count before recursing, increment after (backtrack)
   - Avoids generating duplicate permutations
   
   Example: [1,1,2]
   
   Map: {1: 2, 2: 1}
   
   Start: []
   
   Level 0:
   []
   ├── 1 (freq=2) → [1]
   │   ├── 1 (freq=1) → [1,1]
   │   │   └── 2 (freq=1) → [1,1,2] ✅
   │   └── 2 (freq=1) → [1,2]
   │       └── 1 (freq=1) → [1,2,1] ✅
   │
   └── 2 (freq=1) → [2]
       └── 1 (freq=2) → [2,1]
           └── 1 (freq=1) → [2,1,1] ✅
   
   Key Steps:
   1. Create frequency map of all elements
   2. For each unique number with count > 0:
      - Choose: Decrement its count
      - Explore: Recurse to next position
      - Unchoose: Increment its count back (backtrack)
   3. When combination size == array size, add to result
   
   Time Complexity: O(n! * n) - but fewer branches due to duplicates
   Space Complexity: O(n!) for storing unique permutations
